ring.geometry

The base idea of the design is creating a ring geometry which tansforms based on the parameters of the inner radius, outer radius and rotation. Whi-le looping through the geometry.parameters, the values of the innerRadius, outerRadius and the sum of both parameters define the xyz translation of every ring. The idividual rotation of each ring is based on its list index value in the ring list and the rotation parameter. The rings can be scaled by a z factor. To qualify it a useful tool for a potential app user, the code creates a huge variety of geometric constellations with only a small change in parameters. Therefore every paramteter can be controlled via the app slider interface.The code itself contains at the beginning the inclusion of the necessary libraries, the creation of the empty lists, and in the following the dictionary for the geometry parameters. In the first for loop a ring is created, based on the geometry parameters of the dictionary. To make it possible for a user to change the geometry, two conditional statementent are necessary. The first one checks if the number of geometries, controlled by the user is not equal to zero. The second statement checks if the lenght of the current number of rings is equal to the current number of geometries that the user defines. In this case, the current number and configuration doesnâ€˜t change. Other-wise, the old geometries are removed and the new rings will be created.
